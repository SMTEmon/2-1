# Quick Review: Code Smells & Refactoring

## 1. Code Smells
Indicators of deeper problems (not bugs). They hurt maintainability.

### The 5 Categories
1. **Bloaters:** Code that grew too large.
   * *Long Method, Large Class, Primitive Obsession, Long Param List, Data Clumps.*
2. **OO Abusers:** Misusing object-oriented principles.
   * *Switch Statements, Temporary Field, Refused Bequest, Alt Classes w/ Diff Interfaces.*
3. **Change Preventers:** Hard to modify.
   * *Divergent Change, Shotgun Surgery, Parallel Inheritance Hierarchies.*
4. **Dispensable:** Unnecessary code.
   * *Comments, Duplicate Code, Lazy Class, Data Class, Dead Code, Speculative Generality.*
5. **Couplers:** Excessive coupling.
   * *Feature Envy, Inappropriate Intimacy, Message Chains, Middle Man.*

## 2. Refactoring Techniques
Restructuring code **without** changing behavior.

| Technique | When to Use |
|-----------|-------------|
| **Extract Method** | Method is too long or complex. |
| **Inline Method** | Method body is trivial; name adds no value. |
| **Extract Class** | One class does the work of two. |
| **Move Method/Field** | Method/field is used more by another class. |
| **Rename** | Name doesn't reveal purpose. |
| **Replace Temp with Query** | Temp variable stores a result of an expression. |
| **Introduce Param Object** | Long list of parameters. |
| **Replace Switch w/ Polymorphism** | Switch/if-else based on type codes. |
| **Replace Magic Number** | Unexplained literal numbers. |
| **Encapsulate Field** | Public fields accessed directly. |

## 3. Key Principles

### Law of Demeter
**"Don't talk to strangers."** Only talk to immediate friends (self, fields, args).
*   ❌ `a.getB().getC().doWork()`
*   ✅ `a.doWork()` (via delegation)

### Middle Man
A class that only delegates.
*   **Bad:** Adds complexity for no reason.
*   **Good:** If it's a **Proxy**, **Decorator**, or **Adapter** (adds value).
