***

> [!danger] The 10-Second Summary
> **Generics** allow types (classes, interfaces, methods) to be **parameters**, providing **compile-time type safety**. Eliminates `ClassCastException` and manual casting.

---

### 1. The Core Problem vs. The Solution
*   ❌ **Without Generics:** `List list = new ArrayList();` — You can add anything. Type errors only show up at **runtime** as `ClassCastException`. Requires manual casting.
*   ✅ **With Generics:** `List<String> list = new ArrayList<>();` — Compiler rejects wrong types **before** the program runs. No casting needed.

---

### 2. Generic Classes & Methods
*   **Generic Class:** `class Box<T>` — `T` is a placeholder. User decides the type: `Box<Integer>` or `Box<String>`.
*   **Generic Method:** `<T> void print(T[] arr)` — The `<T>` goes **before** the return type.
*   **Bounded Types:** `<T extends Number>` — Restricts `T` to Number or its subclasses. Gives access to methods like `.doubleValue()`.

---

### 3. Wildcards (`?`) — The Big Exam Topic
| Wildcard | Meaning | Can Read? | Can Add? | Use When... |
|----------|---------|-----------|----------|-------------|
| `?` | Any unknown type | Yes (as `Object`) | No (except `null`) | Just iterating/printing |
| `? extends Type` | Type or subclass | Yes (as `Type`) | No | **Reading** data (Producer) |
| `? super Type` | Type or superclass | Yes (as `Object`) | Yes | **Writing** data (Consumer) |

> [!tip] PECS Mnemonic
> **P**roducer **E**xtends, **C**onsumer **S**uper.
> * Need to **GET** items? Use `extends`.
> * Need to **ADD** items? Use `super`.

### `?` vs `T`
*   Use `?` when you just need to **read** and don't care about the specific type.
*   Use `T` when you need to **link** types (e.g., return the same type you received, or add items).

---

### 4. Type Erasure
*   **What:** Java **removes** all generic type info at runtime. `<String>` becomes raw `List`.
*   **Why:** Backward compatibility with pre-Java 5 code.
*   **Result:** `List<String>` and `List<Integer>` are the **same class** at runtime.
*   **Restrictions:** You **cannot** do `new T()`, `new T[]`, or `instanceof List<String>`.

---

### 5. Generic Interfaces
Just like classes, interfaces can be generic: `interface Pair<K, V>`. Implementing classes either specify the types (`implements Pair<String, Integer>`) or stay generic (`class MyPair<K, V> implements Pair<K, V>`).

---

### 🧠 Quick Code Trigger for the Exam

```java
// Generic Class
class Box<T> {
    private T value;
    public void set(T v) { value = v; }
    public T get() { return value; }
}

// Bounded Type Parameter
class MathBox<T extends Number> {
    double doubled() { return value.doubleValue() * 2; }
}

// Wildcard Examples
void printList(List<?> list) { /* read only */ }
double sum(List<? extends Number> list) { /* read as Number */ }
void addInts(List<? super Integer> list) { /* can add Integer */ }
```
