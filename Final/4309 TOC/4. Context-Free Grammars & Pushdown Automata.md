## 1. The Chomsky Hierarchy (Context)
Before diving into CFGs, it helps to understand where they fit in the grand scheme of formal languages. The **Chomsky Hierarchy** categorizes formal grammars into four levels of increasing power:

1.  **Regular Languages (Type 3):** Recognized by Finite Automata (DFA/NFA) and Regular Expressions.
2.  **Context-Free Languages (Type 2):** Recognized by Pushdown Automata (PDA) and Context-Free Grammars (CFG). *(Focus of this note)*.
3.  **Context-Sensitive Languages (Type 1):** Recognized by Linear Bounded Automata.
4.  **Recursively Enumerable / Turing Recognizable (Type 0):** Recognized by Turing Machines.

$$Regular \subset Context\text{-}Free \subset Turing\text{-}Decidable \subset Turing\text{-}Recognizable$$

## 2. Context-Free Grammars (CFG)

> [!definition] What is a CFG?
> In formal language theory, a **Context-Free Grammar (CFG)** is a formal grammar whose production rules can be applied to a nonterminal symbol *regardless of its context* (i.e., regardless of the symbols surrounding it).

### Formal Definition
A CFG is defined as a 4-tuple $G = (V, \Sigma, R, S)$:
*   $V$: Finite set of **Variables** (or non-terminals). These appear on the left-hand side of rules. Usually uppercase letters (e.g., $S, A, B$).
*   $\Sigma$: Finite set of **Terminals** (alphabet). These appear only on the right-hand side. Usually lowercase letters, numbers, or symbols (e.g., `0, 1, a, +, ( `).
*   $R$: Finite set of **Rules** (or productions). Format: $V \rightarrow (V \cup \Sigma)^*$.
*   $S$: **Start Variable** ($S \in V$). The top-left symbol where derivations begin.

### How Grammars Generate Strings
1.  Write down the start variable $S$.
2.  Replace any variable according to a valid rule.
3.  Repeat step 2 until only terminals remain.
4.  The result is the generated string.
5.  **Language of a Grammar $L(G)$:** The set of all strings generated by $G$.

**Derivation Notation:**
*   $u \Rightarrow v$: $u$ yields $v$ in *one* substitution step.
*   $u \Rightarrow^* v$: $u$ yields $v$ in *zero or more* substitution steps.
*   If $S \Rightarrow^* w$ (where $w \in \Sigma^*$), this sequence is called a **derivation** of $w$.

### Example
Let $L = \{0^k 1^k \mid k \geq 0\}$.
**CFG Rules:**
$S \rightarrow 0S1$
$S \rightarrow R$
$R \rightarrow \epsilon$

**Derivation of `0011`:**
$S \Rightarrow 0S1 \Rightarrow 00S11 \Rightarrow 00R11 \Rightarrow 00\epsilon11 \Rightarrow 0011$

## 3. Derivations and Parse Trees

A **Parse Tree** is a visual representation of a derivation, abstracting away the order in which variables were substituted.
*   **Root:** Start variable $S$.
*   **Internal Nodes:** Variables ($V$).
*   **Leaves:** Terminals ($\Sigma$) or $\epsilon$.

### Types of Derivations
Because we can choose any variable to replace at any time, we formalize derivations into two standard types:
1.  **Leftmost Derivation:** At each step, the *leftmost* variable in the sentential form is replaced.
2.  **Rightmost (Canonical) Derivation:** At each step, the *rightmost* variable is replaced.

## 4. Ambiguity

> [!warning] Definition of Ambiguity
> A grammar is **ambiguous** if there exists *at least one string* in the language that has **two or more distinct parse trees** (which is equivalent to having two or more distinct leftmost derivations or rightmost derivations).

**Real-world analogy:** "The boy saw the girl with the mirror."
*   *Meaning 1:* The boy used a mirror to see the girl.
*   *Meaning 2:* The boy saw a girl who was holding a mirror.

### CFG Example of Ambiguity
Consider grammar $G_3$: $E \rightarrow E+E \mid E \times E \mid (E) \mid a$
String: $a + a \times a$

This string has two parse trees depending on precedence:
1.  `(a + a) * a` (Addition happens first)
2.  `a + (a * a)` (Multiplication happens first - usually the mathematically correct one)

*Note on resolving ambiguity:* To fix $G_3$, we introduce hierarchy/precedence layers (e.g., $E \to E+T \mid T$ and $T \to T \times F \mid F$), which forces multiplication to happen lower in the tree (resolved earlier).

> [!info] Inherent Ambiguity (Internet Supplement)
> While some ambiguous grammars can be rewritten into unambiguous ones (like the math example), some languages are **inherently ambiguous**. This means *every* possible CFG that generates the language is ambiguous. Example: $L = \{a^n b^n c^m\} \cup \{a^n b^m c^m\}$.

## 5. Pushdown Automata (PDA)

> [!definition] What is a PDA?
> A **Pushdown Automaton** is essentially a Finite Automaton (NFA/DFA) equipped with an infinite memory capacity in the form of a **Stack** (LIFO - Last In, First Out).

*   **Read:** Input from a tape.
*   **Push:** Add symbols to the top of the stack.
*   **Pop:** Remove symbols from the top of the stack.

### Formal Definition
A PDA is a 6-tuple $M = (Q, \Sigma, \Gamma, \delta, q_0, F)$:
*   $Q$: Finite set of states.
*   $\Sigma$: Input alphabet.
*   $\Gamma$: **Stack alphabet** (symbols allowed on the stack).
*   $\delta$: Transition function. $\delta: Q \times \Sigma_\epsilon \times \Gamma_\epsilon \rightarrow \mathcal{P}(Q \times \Gamma_\epsilon)$
*   $q_0$: Start state.
*   $F$: Set of accept states.

> [!note] Transition Notation
> In schematic diagrams, an arrow labeled $a, b \rightarrow c$ means:
> "If you read `a` from the input and pop `b` from the top of the stack, then push `c` onto the stack."
> *(Note: $\epsilon$ can be used to represent "do not read/pop/push anything").*

### Determinism vs. Non-determinism
*   Unlike Finite Automata (where DFA and NFA have equal power), **Non-deterministic PDAs (NPDA) are strictly more powerful than Deterministic PDAs (DPDA).**
*   Our standard PDA model is *non-deterministic*. Example: Recognizing palindromes $ww^R$ requires guessing where the middle of the string is, splitting threads to replicate the stack.

## 6. Equivalence of CFG and PDA

> [!abstract] Theorem
> A language $A$ is a Context-Free Language (CFL) **if and only if** some Pushdown Automaton recognizes it. ($CFL \iff PDA$).

### Proof Intuition (CFG $\to$ PDA)
To convert a CFG to a PDA, the PDA simulates the **top-down leftmost derivation** of the grammar:
1.  Push the start symbol $S$ onto the stack.
2.  **If the top of the stack is a Variable ($A$):** Nondeterministically guess a rule $A \rightarrow u$ and replace $A$ with the string $u$ on the stack.
3.  **If the top of the stack is a Terminal ($a$):** Pop it and match it with the next symbol from the input tape. If they don't match, that thread rejects.
4.  If the stack is empty and the input is fully read, **Accept**.

## 7. Chomsky Normal Form (CNF)

CNF is a standardized, simplified form for Context-Free Grammars. It is highly useful for algorithms (like CYK) and parsing proofs.

> [!definition] CNF Rule Structure
> A CFG is in Chomsky Normal Form if **every** rule is of the form:
> 1. $A \rightarrow BC$ (Two variables)
> 2. $A \rightarrow a$ (One terminal)
> 
> *Exceptions/Rules:* $B$ and $C$ cannot be the start variable. The rule $S \rightarrow \epsilon$ is permitted only if $S$ is the start variable.

### 4 Steps to Convert any CFG to CNF
*(Order must be maintained strictly)*

1.  **New Start Symbol:**
    *   Add a new start variable $S_0$ and rule $S_0 \rightarrow S$. (Ensures the start variable never appears on the right side).
2.  **Eliminate $\epsilon$-rules:**
    *   Remove any $A \rightarrow \epsilon$ (where $A \neq S_0$).
    *   For every rule containing $A$ (e.g., $R \rightarrow uAv$), add a rule without $A$ ($R \rightarrow uv$). If $A$ appears multiple times, add all combinations.
3.  **Eliminate Unit Rules:**
    *   Remove rules of the form $A \rightarrow B$.
    *   Whenever $B \rightarrow u$ exists, add $A \rightarrow u$. Repeat until all unit rules are gone.
4.  **Convert Remaining Rules (Binarization & Terminal Replacement):**
    *   *Binarization:* Replace long rules $A \rightarrow u_1 u_2 ... u_k$ (where $k \ge 3$) with a chain: $A \rightarrow u_1 A_1$, $A_1 \rightarrow u_2 A_2$, ..., $A_{k-2} \rightarrow u_{k-1} u_k$.
    *   *Terminal Replacement:* Replace any terminal in a mixed/multi-symbol rule with a new variable (e.g., replace `a` with $U_a$ and add $U_a \rightarrow a$).

## 8. Cocke–Younger–Kasami (CYK) Algorithm

> [!info] Purpose
> The CYK algorithm is a dynamic programming parsing algorithm used to determine if a string $w$ belongs to the language $L(G)$ of a given CFG.

*   **Prerequisite:** The grammar $G$ **must** be in Chomsky Normal Form (CNF).
*   **Time Complexity:** $\mathcal{O}(n^3 \cdot |G|)$ where $n$ is string length, $|G|$ is grammar size.
*   **Space Complexity:** $\mathcal{O}(n^2)$ to store the triangular table.

### Algorithm Steps
Given string $w = s_1 s_2 \dots s_n$:
We construct an $n \times n$ triangular table $T$. Cell $T[i,j]$ contains the set of variables that can generate the substring starting at index $i$ with length $j$.

1.  **Base Case (Length $j=1$):**
    For each position $i$ (from 1 to $n$), look at the single character $s_i$.
    $T[i,1] = \{A \mid A \rightarrow s_i \in R\}$
2.  **Recursive Step (Length $j > 1$):**
    To find $T[i,j]$, split the substring into two parts at every possible split point $k$ ($1 \le k < j$).
    Look for rules $A \rightarrow BC$ where:
    *   $B \in T[i, k]$ (Prefix of length $k$)
    *   $C \in T[i+k, j-k]$ (Suffix of length $j-k$)
    Add $A$ to $T[i,j]$.
3.  **Acceptance:**
    The string $w$ is accepted if the **Start Variable ($S$)** is in the top-right cell: $S \in T[1, n]$.

## 9. Pumping Lemma for Context-Free Languages

Just as the regular pumping lemma proves a language is not regular, the CFL pumping lemma is used to prove a language is **not Context-Free**.

> [!abstract] Pumping Lemma for CFLs
> For every Context-Free Language $A$, there exists a pumping length $p$ such that if $s \in A$ and $|s| \ge p$, then $s$ can be divided into five pieces $s = uvxyz$ satisfying three conditions:
> 1. $uv^i x y^i z \in A$ for all $i \ge 0$ (The string remains in the language when pumped).
> 2. $|vy| \ge 1$ (or $vy \neq \epsilon$) (At least one of $v$ or $y$ must not be empty).
> 3. $|vxy| \le p$ (The pumped portion is not too spread out).

### Proof by Picture (Intuition)
If a string is extremely long ($|s| \ge p$), its parse tree must be very tall.
Because the grammar has a finite number of variables $|V|$, any sufficiently long path from the root to a leaf *must* repeat a variable (Pigeonhole Principle).
Let $R$ be the repeating variable.
*   The top $R$ generates $u(R)z$.
*   The bottom $R$ generates $v(x)y$.
By "cutting and pasting" the tree at variable $R$, we can pump $v$ and $y$ up (repeating the middle segment) or down (removing it), generating $uv^2xy^2z$, $uxz$, etc.

### Example: Proving $B = \{0^n 1^n 2^n \mid n \ge 0\}$ is NOT a CFL
1.  **Assume for contradiction** that $B$ is a CFL.
2.  Let $p$ be the pumping length. Choose string $s = 0^p 1^p 2^p \in B$.
3.  The lemma states $s = uvxyz$ satisfying the three conditions.
4.  Look at Condition 3: $|vxy| \le p$.
    *   Because the length of $vxy$ is at most $p$, the substring $vxy$ can span across *at most two* distinct symbols (e.g., only 0s and 1s, or only 1s and 2s). It cannot contain 0s, 1s, AND 2s simultaneously.
5.  Look at Condition 1: $uv^2xy^2z \in B$.
    *   When we pump $v$ and $y$, we are adding more of some symbols, but *not* all three.
    *   Therefore, the balance of equal 0s, 1s, and 2s is broken.
6.  The pumped string is $\notin B$, violating Condition 1.
7.  **Contradiction!** Thus, $B$ is not a CFL.
