### 1. TM Basics & 7-Tuple (Memorize this)
> [!info] $M = (Q, \Sigma, \Gamma, \delta, q_0, q_{acc}, q_{rej})$
- **$\Sigma$**: Input alphabet (NO blanks `Ë˝`).
- **$\Gamma$**: Tape alphabet ($\Sigma \subset \Gamma$, includes blanks `Ë˝`).
- **$\delta$**: $Q \times \Gamma \rightarrow Q \times \Gamma \times \{L, R\}$ *(State + Read $\rightarrow$ State + Write + Move)*
- **3 Outcomes:** Accept, Reject (by halting), **Reject (by looping)**.

### 2. Recognizable vs. Decidable (Crucial Distinction)
- **Turing-Recognizable (TM):** If input is valid, TM accepts. If invalid, TM might reject OR **loop forever**. 
- **Turing-Decidable (Decider):** TM **ALWAYS HALTS**. It will definitively Accept or Reject. No infinite loops.
- *Hierarchy:* Regular $\subset$ Context-Free $\subset$ Decidable $\subset$ Recognizable

### 3. TM Variants (All are EQUIVALENT to standard TM)
*Exam Tip: Know HOW the standard TM simulates them.*
1. **Multi-Tape $\rightarrow$ Single Tape:**
   - Tape content separated by `#`. 
   - Virtual heads tracked using **dotted symbols** (e.g., $\dot{a}$).
2. **Nondeterministic (NTM) $\rightarrow$ Deterministic (DTM):**
   - DTM simulates the NTM's computation tree using **Breadth-First Search (BFS)**.
   - *Why BFS?* DFS might get stuck in an infinite loop branch!
3. **Enumerator:**
   - Prints valid strings. Equivalent to a TM via **dovetailing** (running multiple inputs step-by-step concurrently to avoid looping traps).

### 4. Church-Turing Thesis
- **Concept:** Intuitive algorithm = Formal Turing Machine. 
- **Exam Application:** This gives you permission to write TM proofs using "high-level English algorithms" instead of drawing state diagrams.

### 5. Decidable Problems (The "Yes" List)
*How to prove they are decidable? Build a TM that does the following:*
- **$A_{DFA}$ (Accept DFA):** Just simulate the DFA.
- **$A_{NFA}$ (Accept NFA):** Convert NFA to DFA $\rightarrow$ run $A_{DFA}$.
- **$E_{DFA}$ (Empty DFA):** Mark start state $\rightarrow$ trace paths $\rightarrow$ accept if NO accept state is marked.
- **$EQ_{DFA}$ (Equivalent DFAs):** Construct a DFA for the **Symmetric Difference**: $(A \cap \overline{B}) \cup (\overline{A} \cap B)$. Run $E_{DFA}$. If empty, they are equivalent.
- **$A_{CFG}$ (Accept CFG):** Convert to **Chomsky Normal Form (CNF)**. Test all derivations of length **$2|w| - 1$**.

### 6. The "No" List & The Universal TM
> [!warning] $A_{TM}$ (Acceptance Problem for TMs)
> $A_{TM} = \{\langle M, w \rangle \mid M \text{ accepts } w\}$
- **Is it Recognizable?** YES. By the **Universal Turing Machine (UTM)** $U$, which just simulates $M$ on $w$.
- **Is it Decidable?** NO. If $M$ loops, $U$ loops. We cannot force it to halt.

### 7. THE HALTING PROBLEM (Guaranteed Exam Question)
> [!danger] $HALT_{TM}$ is UNDECIDABLE.
> **Proof by Contradiction (Diagonalization Paradox)**

**Memorize these 4 steps for the exam:**
1. **Assume** a decider $H$ exists for $HALT_{TM}$. 
   *($H$ returns ACCEPT if machine halts, REJECT if machine loops).*
2. **Construct** a malicious TM, $D$ (the Deceiver), which takes $H$ as a subroutine.
3. **The Rule of $D$:**
   - Run $H$ to ask: "Does machine $M$ halt on its own code $\langle M \rangle$?"
   - **Do the exact opposite of $H$:**
     - If $H$ says "Halts" $\rightarrow$ $D$ intentionally infinite loops.
     - If $H$ says "Loops" $\rightarrow$ $D$ immediately halts & accepts.
4. **The Paradox:** Feed $D$ into $D$ (i.e., run $D(\langle D \rangle)$).
   - If $D$ halts, $H$ says it halts, so $D$ loops. 
   - If $D$ loops, $H$ says it loops, so $D$ halts.
   - **Contradiction!** Therefore, $H$ cannot exist. The Halting problem is undecidable.
