# 📚 Binary Heap & Priority Queue
**Tags:** #DataStructures #Heaps #PriorityQueue #ExamReview 

> [!summary] Why do we need Heaps?
> Arrays and Linked Lists hit performance bottlenecks for Priority Queues (PQ).
> - **Unsorted Array:** Insert `O(1)`, Extract Max `O(N)` (Too slow to find)
> - **Sorted Array:** Insert `O(N)` (Too slow to add), Extract Max `O(1)`
> - **Binary Heap:** Insert ==`O(log N)`==, Extract Max ==`O(log N)`==

---

## 1. What is a Binary Heap?
A binary tree that strictly follows **two properties**:
1. **Heap Property (Max-Heap):** `Parent(i) >= Children(i)`. 
   - *Result:* The largest element is always at the root.
   - *(Note: Min-Heap is exact opposite: `Parent(i) <= Children(i)`)*
2. **Shape Property:** It is a **Complete Binary Tree**.
   - All levels are fully filled except possibly the last.
   - Last level nodes are filled strictly from **left to right**.

> [!example]- Visual: Valid Max Heap
> ```mermaid
> graph TD
>     A((90)) --> B((85))
>     A --> C((80))
>     B --> D((75))
>     B --> E((72))
>     C --> F((74))
>     C --> G((78))
> ```

## 2. Array Implementation (The Standard)
Heaps are implicitly represented using an array (Level Order traversal). No pointers needed!
* **Next free spot** is always at `index = size + 1`.

### Navigation Formulas (1-Based Indexing)
If a node is at index $i$:
* **Left Child:** $2i$
* **Right Child:** $2i + 1$
* **Parent:** $\lfloor i/2 \rfloor$

> [!important] The Leaf Node Property
> In a heap of size $N$:
> - **Internal Nodes:** Indices $1$ to $\lfloor N/2 \rfloor$
> - **Leaf Nodes:** Indices $\lfloor N/2 \rfloor + 1$ to $N$

---

## 3. Core Operations

### A. Heapify Down (Fixing a Violation)
Used when a parent is **smaller** than its children. "Demotes" the node down the tree.
* **Rule:** Swap the violating parent with the ==LARGEST== of its two children.
* **Why the largest?** Swapping with the smaller child would still leave the larger child below a smaller parent, keeping the tree broken.
* **Time Complexity:** ==`O(log N)`== (Proportional to tree height).
* ```c++
  max_heapify(i){
	  l = 2 * i;
	  r = 2 * i + 1;
	  largest = i
	  if l <= heapSize && heap[l] > heap[i]
		  largest = l;
	  else if r <= heapSize && heap[r] > heap[largest]
		  largest = r;
	  if largest != i
		  exchange(heap[i] <-> heap[largest]);
		  max(heapify(largest));
  }  
  ```

### B. Build Heap (Unordered Array $\to$ Heap)
* **Naive Top-Down Fails:** Assumes subtrees are already valid. Large values at the bottom never bubble up.
* **Correct Approach (Bottom-Up):** Fix small sub-trees first, then move up.
	* **Start:** Index $\lfloor N/2 \rfloor$ (The last internal node).
	* **Direction:** Iterate backwards down to `1`.
	* **Algorithm:** `for i = N/2 down to 1: max_heapify(i)`
* **Time Complexity:** ==`O(N)` Linear Time==. 
	* *Proof intuition:* Most nodes are leaves (height 0 = 0 work). Only 1 node is at the root (max height = max work). Arithmetico-Geometric series converges to $O(N)$, *not* $O(N \log N)$.

### C. Insert (Percolate Up)
"The new guy has to earn his way to the top."
1. **Shape First:** Insert new key at the end of the array (index `size + 1`).
2. **Order Second:** Compare new key with its Parent.
3. **Swap:** If `Child > Parent`, swap them. Repeat until valid.
* **Time Complexity:** ==`O(log N)`== | **Space:** `O(1)`

### D. Extract Max (Delete Root)
1. **Swap:** Swap Root with the **Last Leaf** (keeps the tree "Complete").
2. **Remove:** Decrement `heap_size` (cuts off the old root).
3. **Fix:** Call `max_heapify(1)` to bubble the new (usually small) root down.
* **Time Complexity:** ==`O(log N)`==

## 4. Advanced Operations

### Arbitrary Deletion (Delete value $X$)
* **Step 1 (The Bottleneck):** Find the node. Because heaps are *weakly ordered*, you can't binary search. Must scan linearly $\to$ **`O(N)`**.
* **Step 2:** Swap with last element, decrease size.
* **Step 3:** Fix the heap:
  * If new node $>$ Parent $\to$ **Percolate Up**.
  * If new node $<$ Children $\to$ **Heapify Down**.
* **Total Time:** `O(N)`. *(Note: If you already know the index, it's `O(log N)`)*.

### Modifying Keys (e.g., Dijkstra's Algorithm)
* **Increase Key:** Value goes UP. Might be larger than parent $\to$ Update value, then **Percolate Up**.
* **Decrease Key:** Value goes DOWN. Might be smaller than children $\to$ Update value, then **Heapify Down**.
* **Time:** `O(log N)` if index is known; otherwise `O(N)` to find it first.

---

## 5. Application: Heap Sort
**Idea:** Extract the maximum element $N-1$ times.
1. Build Max Heap from unordered array $\to$ `O(N)`.
2. Loop $i$ from $N$ down to $2$:
   - Swap Root `A[1]` with Last `A[i]`. (Largest element is now "sorted" at the end).
   - Reduce heap size by 1.
   - Call `max_heapify(1)` to fix the root.
> [!info] Complexity
> - **Time:** ==`O(N log N)`== (Best, Average, and Worst case)
> - **Space:** ==`O(1)`== (In-place sorting)
