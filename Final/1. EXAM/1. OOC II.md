## 🏗️ SOLID Principles
- [ ] **[[2.1 Dependency Injection (DI) & Inversion of Control (IoC)]] 
	- [ ] Understand Inversion of Control.
	- [ ] Create object of another class & ensure loose coupling.
	- [ ] Receive object outside that class (via constructor or setter method).
- [ ] [[Final/4301 OOC2/Detailed/2.2 Liskov Substitution Principle (LSP)|2.2 Liskov Substitution Principle (LSP)]]
	- [ ] Understand parent-child/hierarchical relationships.
	- [ ] Parent and subclasses should be used alternatively.
	- [ ] Subclasses should adhere to all parent behavior (no exceptions should occur).
- [ ] [[Final/4301 OOC2/Detailed/2.3 Interface Segregation Principle (ISP)|2.3 Interface Segregation Principle (ISP)]]
	- [ ] Rule: One interface should perform one responsibility.
